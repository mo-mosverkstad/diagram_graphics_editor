# Diagram and graphics editor

The diagram editor is designed for creating and editing diagrams represented by a graphical descriptive markup syntax that stores the core semantics of the diagram. Unlike explicit vector graphics that stores shapes but does not carry semantic meaning of the diagram, such as SVG and PowerPoint, graphical descriptive markup syntax can also store the semantic relationship. For instance, TikZ can store the necessary geometrical structures. Additionally, self-defined syntax is also allowed, for example one describing an electrical circuit in schematics, a hardware circuit with complex boolean logic or biochemical reaction pathways. This allows simplicity and flexibility when creating and modifying the diagram graphically.
